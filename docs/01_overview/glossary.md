# 📖 용어집 (Glossary)

## 📋 개요

이 문서는 CoUp 프로젝트에서 사용되는 도메인 용어와 기술 용어를 정의합니다. 다른 문서를 읽다가 모르는 용어가 나오면 이 문서를 참조하세요.

---

## 🏷️ 도메인 용어

### 사용자 관련

| 용어 | 영문 | 설명 |
|------|------|------|
| **사용자** | User | 서비스에 가입한 계정 |
| **일반 사용자** | USER | 기본 역할, 스터디 참여 가능 |
| **관리자** | ADMIN | 시스템 관리 권한 보유 |
| **슈퍼 관리자** | SUPER_ADMIN | 최고 관리 권한 보유 |
| **스터디장** | Owner | 스터디를 생성한 사람 |
| **스터디 관리자** | Manager | 스터디 운영 권한 보유 멤버 |
| **스터디 멤버** | Member | 스터디에 참여 중인 일반 멤버 |

### 스터디 관련

| 용어 | 영문 | 설명 |
|------|------|------|
| **스터디** | Study | 함께 학습하는 그룹 단위 |
| **스터디 멤버십** | StudyMember | 사용자와 스터디의 연결 관계 |
| **가입 신청** | Join Request | 스터디 가입을 위한 요청 |
| **공개 스터디** | Public Study | 누구나 검색 가능한 스터디 |
| **비공개 스터디** | Private Study | 초대로만 가입 가능한 스터디 |
| **카테고리** | Category | 스터디 분류 (프로그래밍, 어학 등) |

### 그룹 관련

| 용어 | 영문 | 설명 |
|------|------|------|
| **그룹** | Group | 스터디 내 소그룹 |
| **그룹 멤버** | GroupMember | 그룹에 속한 멤버 |
| **그룹 초대** | GroupInvite | 그룹 가입 초대 |

### 활동 관련

| 용어 | 영문 | 설명 |
|------|------|------|
| **메시지** | Message | 채팅 메시지 |
| **태스크** | Task | 할 일, 과제 |
| **담당자** | Assignee | 태스크에 배정된 사람 |
| **이벤트** | Event | 캘린더 일정 |
| **공지** | Notice | 스터디 공지사항 |
| **알림** | Notification | 시스템 알림 |

### 상태값

| 용어 | 영문 | 설명 |
|------|------|------|
| **활성** | ACTIVE | 정상 상태 |
| **정지** | SUSPENDED | 일시 정지 상태 |
| **삭제됨** | DELETED | 삭제된 상태 |
| **대기 중** | PENDING | 승인 대기 중 |
| **승인됨** | APPROVED | 승인 완료 |
| **거절됨** | REJECTED | 승인 거절 |

### 태스크 상태

| 용어 | 영문 | 설명 |
|------|------|------|
| **할 일** | TODO | 아직 시작하지 않음 |
| **진행 중** | IN_PROGRESS | 작업 중 |
| **완료** | DONE | 작업 완료 |

## 💻 기술 용어

### Next.js 관련

| 용어 | 설명 |
|------|------|
| **App Router** | Next.js 13+의 새로운 라우팅 시스템, 폴더 기반 라우팅 |
| **Page Router** | 기존 pages/ 폴더 기반 라우팅 (이 프로젝트에서는 미사용) |
| **서버 컴포넌트** | 서버에서만 렌더링되는 React 컴포넌트, 기본값 |
| **클라이언트 컴포넌트** | 브라우저에서 렌더링되는 컴포넌트, 'use client' 지시어 필요 |
| **API Route** | /api 경로의 백엔드 엔드포인트 |
| **Middleware** | 요청 처리 전 실행되는 코드 (인증 체크 등) |
| **라우트 그룹** | 괄호로 감싼 폴더, URL에 포함되지 않음 예: (auth) |
| **동적 라우트** | 대괄호 폴더, URL 파라미터 예: [id] |

### 렌더링 관련

| 용어 | 설명 |
|------|------|
| **SSR** | Server-Side Rendering, 서버에서 HTML 생성 |
| **CSR** | Client-Side Rendering, 브라우저에서 렌더링 |
| **SSG** | Static Site Generation, 빌드 시 HTML 생성 |
| **ISR** | Incremental Static Regeneration, 점진적 정적 재생성 |
| **Hydration** | 서버 렌더링된 HTML에 이벤트 핸들러 연결 |

### 데이터베이스 관련

| 용어 | 설명 |
|------|------|
| **ORM** | Object-Relational Mapping, 객체-관계 매핑 |
| **Prisma** | Node.js/TypeScript용 ORM |
| **스키마** | 데이터베이스 구조 정의 |
| **마이그레이션** | 스키마 변경을 데이터베이스에 적용 |
| **시딩** | 초기 데이터 삽입 |
| **인덱스** | 검색 속도 향상을 위한 데이터 구조 |
| **외래 키** | 다른 테이블을 참조하는 키 |
| **관계** | 테이블 간 연결 (1:1, 1:N, N:M) |

### 인증 관련

| 용어 | 설명 |
|------|------|
| **인증** | Authentication, 사용자가 누구인지 확인 |
| **인가** | Authorization, 사용자가 무엇을 할 수 있는지 결정 |
| **세션** | 로그인 상태 유지를 위한 서버 저장 정보 |
| **JWT** | JSON Web Token, 토큰 기반 인증 |
| **OAuth** | 소셜 로그인 프로토콜 |
| **RBAC** | Role-Based Access Control, 역할 기반 접근 제어 |

### 상태 관리 관련

| 용어 | 설명 |
|------|------|
| **상태** | State, 컴포넌트의 데이터 |
| **전역 상태** | 여러 컴포넌트에서 공유하는 상태 |
| **서버 상태** | 서버에서 가져온 데이터 (캐싱 대상) |
| **클라이언트 상태** | 클라이언트에서만 존재하는 상태 |
| **Context** | React의 전역 상태 관리 방법 |
| **캐싱** | 데이터를 저장해두고 재사용 |
| **쿼리** | 데이터 조회 요청 |
| **뮤테이션** | 데이터 변경 요청 |

### 실시간 통신 관련

| 용어 | 설명 |
|------|------|
| **WebSocket** | 양방향 실시간 통신 프로토콜 |
| **Socket.io** | WebSocket 기반 실시간 통신 라이브러리 |
| **이벤트** | 소켓으로 주고받는 메시지 단위 |
| **룸** | 특정 클라이언트 그룹에게만 메시지 전송 |
| **Redis Adapter** | 여러 서버 간 소켓 이벤트 동기화 |

### 테스팅 관련

| 용어 | 설명 |
|------|------|
| **단위 테스트** | 개별 함수/컴포넌트 테스트 |
| **통합 테스트** | 여러 모듈 간 상호작용 테스트 |
| **E2E 테스트** | 전체 사용자 플로우 테스트 |
| **모킹** | 가짜 데이터/함수로 대체 |
| **커버리지** | 테스트가 코드를 얼마나 검사하는지 비율 |

### API 관련

| 용어 | 설명 |
|------|------|
| **REST** | Representational State Transfer, API 설계 스타일 |
| **엔드포인트** | API 요청 URL |
| **HTTP 메서드** | GET, POST, PATCH, DELETE 등 |
| **요청 본문** | Request Body, 요청 시 전송하는 데이터 |
| **응답** | Response, 서버가 반환하는 데이터 |
| **상태 코드** | 200 성공, 400 클라이언트 오류, 500 서버 오류 |

## 🔤 약어

| 약어 | 전체 | 설명 |
|------|------|------|
| **API** | Application Programming Interface | 프로그램 간 통신 인터페이스 |
| **URL** | Uniform Resource Locator | 웹 주소 |
| **UI** | User Interface | 사용자 인터페이스 |
| **UX** | User Experience | 사용자 경험 |
| **DB** | Database | 데이터베이스 |
| **ORM** | Object-Relational Mapping | 객체-관계 매핑 |
| **JWT** | JSON Web Token | 인증 토큰 |
| **SSR** | Server-Side Rendering | 서버 사이드 렌더링 |
| **CSR** | Client-Side Rendering | 클라이언트 사이드 렌더링 |
| **CRUD** | Create, Read, Update, Delete | 기본 데이터 작업 |
| **XSS** | Cross-Site Scripting | 보안 취약점 유형 |
| **CSRF** | Cross-Site Request Forgery | 보안 취약점 유형 |
