## 🚨 보안 취약점 신고

### 신고 방법

보안 취약점을 발견하셨다면, **공개 이슈를 생성하지 마시고** 비공개로 신고해주세요.

**신고 채널**:
- 📧 이메일: [보안 담당 이메일]
- 📋 GitHub Security Advisories (비공개)

### 신고 내용

다음 정보를 포함해주세요:

1. **취약점 유형**: XSS, SQL Injection, CSRF 등
2. **영향 범위**: 영향을 받는 버전, 기능
3. **재현 방법**: 단계별 재현 절차
4. **증거**: 스크린샷, 로그 등
5. **제안 해결책**: (선택사항)

### 대응 절차

1. **접수 확인**: 48시간 내 접수 확인
2. **조사**: 취약점 분석 및 영향도 평가
3. **수정**: 패치 개발
4. **공개**: 수정 후 보안 권고문 발행

### 보상

- 유효한 보안 취약점 신고에 대해 크레딧을 제공합니다
- SECURITY-ACKNOWLEDGMENTS.md에 이름 등재 (동의 시)

## 🛡️ 보안 기능

### 인증 및 인가
- NextAuth.js 기반 보안 인증
- bcrypt 비밀번호 해싱
- JWT 토큰 기반 세션
- RBAC (역할 기반 접근 제어)

### 데이터 보호
- HTTPS 필수
- SQL Injection 방지 (Prisma ORM)
- XSS 방지 (sanitize-html)
- CSRF 토큰

### 인프라 보안
- 환경 변수 분리
- 민감 정보 암호화
- 보안 헤더 적용

---

## ⚖️ 책임 한계

본 소프트웨어는 "있는 그대로" 제공되며, 보안에 대한 명시적 또는 묵시적 보증을 제공하지 않습니다.

운영 환경에서 사용 시:
- 정기적인 보안 감사 수행
- 최신 버전 유지
- 환경 변수 적절히 설정
- 보안 모범 사례 적용
